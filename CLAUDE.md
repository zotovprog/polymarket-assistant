# CLAUDE.md — Правила разработки

## Перед коммитом (ОБЯЗАТЕЛЬНО)

1. **Запусти тесты**: `python3 -m pytest tests/ -x -q` — все должны пройти
2. **Если тест упал** — разберись ПОЧЕМУ, не правь тест под новый код вслепую
3. **Проверь синтаксис** изменённых файлов: `python3 -c "import ast; ast.parse(open('file.py').read())"`
4. **git diff** — перечитай свои изменения перед коммитом

## Новый функционал

- Если функционал можно включить/выключить — **обязательно** эндпоинты start/stop + state
- **Логирование**: каждое значимое действие пишет лог (INFO для действий, WARNING для проблем, DEBUG для деталей)
- **Обработка ошибок**: все внешние вызовы (API, on-chain) обёрнуты в try/except с логированием
- **Dry run**: для опасных операций (liquidate, redeem) — параметр `dry_run=True` по умолчанию

## Pair Arb специфика

- **Минимум PM**: 5 шар на ордер, $1.00 notional
- **Balance cap**: никогда не ставить больше 40% баланса на одну пару
- **Обе ноги**: проверять баланс МЕЖДУ размещением первой и второй ноги
- **Orphan cleanup**: если одна нога не прошла — отменить другую немедленно
- **Live vs Paper**: по умолчанию paper_mode=True, live только явно
